# Resume Variant Management

When applying to multiple roles, you maintain multiple tailored resumes. This document tracks principles and prevents drift.

## Principles

1. **Ground truth is always the source.** Every variant starts from your ground truth file -- never from another variant.
2. **Variants diverge on emphasis, not on facts.** Tailoring = reordering bullets, selecting which to include, adjusting keyword density. NOT inventing new claims.
3. **One directory per application.** Each contains its own resume, cover letter, PDFs, JD, and README.
4. **No cross-pollination.** When editing Variant B, do NOT copy bullets from Variant A. Go back to ground truth and re-tailor.
5. **Shared fixes propagate forward.** If a correction applies to all variants (e.g., contact info change), update ground truth first, then rebuild active variants. Don't retroactively fix submitted applications.

## Tailoring dimensions

When creating a new variant, decide on these axes:
- **Professional Summary:** which 2-3 strengths to lead with
- **Experience bullets:** which of your available bullets to include, which keywords to front-load
- **Research description:** full narrative vs. compressed (depends on role's research emphasis)
- **Project bullets:** same facts, different keyword emphasis
- **Technical Skills:** same allowed list, but ordering and grouping shift per JD
- **Cover letter framing:** which angle of your research maps best to this role

## Terminology alignment

Some JDs say "AI" exclusively; others say "LLM" or "ML."

Don't do a global find-and-replace without checking each instance. Flag the mismatch, propose the swap, but decide which instances to change on a case-by-case basis.

## Framing pivots

The same role and experience can be framed differently depending on the target:

| Research resume | Content/Design resume |
|---|---|
| evaluation frameworks / statistical analysis | content quality systems / prompt evaluation |
| experimental design / hypothesis testing | editorial QA governance |
| model training coordination | naming guidance / voice frameworks |
| publications front-and-center | publications retained but de-emphasized |
